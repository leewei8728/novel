# 提示词库

本文档收集常用的AI辅助写作提示词，可直接复制使用或根据需要修改。

---

## 一、世界观生成

### 1.1 力量体系生成
```
我要写一部科幻修真融合的小说。
请帮我设计一个力量等级体系，要求：
1. 共10个大等级，每级9个小阶
2. 结合基因进化和宇宙能量修炼
3. 每个等级有明确的力量表现（可量化）
4. 说明突破条件（悟道+科技辅助）
5. 说明寿命变化
请以markdown表格形式输出。
```

### 1.2 势力设计
```
基于 @01-worldbuilding/power-system.md 的力量体系，
帮我设计宇宙中的主要势力分布：
1. 5个一级势力（宇宙级别）
2. 每个势力的特点、理念、控制区域
3. 势力之间的关系（同盟/敌对/中立）
4. 每个势力的代表人物等级
```

### 1.3 资源体系
```
帮我设计修炼所需的资源体系：
1. 各等级对应的资源需求
2. 资源的稀有度分级
3. 资源的获取方式
4. 资源与科技的关系（如何用科技提炼/合成）
```

## 二、人物生成

### 2.1 主角生成
```
根据 @02-characters/templates/character-template.md 模板，
为我创建主角档案，基础设定：
- [年龄/背景]
- [性格特点]
- [特殊天赋]
- [初始处境]
请填充完整的角色档案。
```

### 2.2 配角批量生成
```
为主角创建[X]个重要配角，类型包括：
1. 良师益友型 - [功能描述]
2. 竞争对手型 - [功能描述]
3. 红颜知己型 - [功能描述]
4. 神秘引路人型 - [功能描述]
5. 潜在敌人型 - [功能描述]

每个角色要有：
- 独特性格和说话风格
- 与主角的关系发展线
- 自己的目标和动机
```

### 2.3 反派设计
```
帮我设计一个[等级]的反派角色：
1. 他的邪恶不是无缘无故的，要有合理动机
2. 他有自己的行为逻辑和底线
3. 他与主角形成理念上的对立
4. 他有让读者印象深刻的特征
```

## 三、情节生成

### 3.1 大纲生成
```
基于以下世界观和人物设定：
@01-worldbuilding/power-system.md
@01-worldbuilding/universe-structure.md
@02-characters/protagonist.md

为[X]万字的[第X卷]创建章节大纲，每章包含：
- 章节标题
- 核心情节（50字内）
- 推进哪条线索
- 冲突设置
- 章末钩子
```

### 3.2 情节转折设计
```
当前情节：[描述当前状况]
我需要一个意料之外、情理之中的转折，要求：
1. 与之前的伏笔呼应
2. 推动主角成长
3. 提升故事张力
请给出3个不同方向的转折方案。
```

### 3.3 支线情节
```
主线进展到[描述]阶段，
请设计一条支线情节：
1. 与主线有关联但不干扰主线节奏
2. 可以展现配角[名字]的魅力
3. 为后续[某情节]埋下伏笔
4. 字数控制在[X]字左右
```

## 四、场景描写

### 4.1 战斗场景
```
生成一场[等级A] vs [等级B]的战斗场景：
- 地点：[具体场景]
- [角色A]使用[武器/技能]
- [角色B]使用[武器/技能]
- 战斗结果：[胜负/平手/中断]
- 要体现的看点：[以弱胜强/智取/力压等]
- 字数：约[X]字

参考风格：@05-materials/battle-scenes.md
```

### 4.2 修炼/突破场景
```
描写主角突破到[等级]的过程：
- 突破前的铺垫（心境、准备）
- 突破过程中的感悟
- 科技辅助的具体表现
- 突破后的变化
- 字数：约[X]字

注意：要符合 @01-worldbuilding/power-system.md 的设定
```

### 4.3 场景环境
```
描写[地点名称]的环境：
- 时间：[时间]
- 氛围：[压抑/祥和/紧张/神秘等]
- 要为后续[事件]做铺垫
- 融入科技/修真元素
- 字数：约[X]字
```

### 4.4 科技展示
```
描写[科技设备/技术]的使用场景：
- 使用者：[角色]
- 目的：[做什么]
- 要体现的科技感
- 与修真的结合点
- 字数：约[X]字

参考：@05-materials/tech-descriptions.md
```

## 五、对话生成

### 5.1 角色对话
```
生成[角色A]和[角色B]的对话：
- 场景：[在哪里]
- 讨论话题：[什么事]
- 两人关系：[描述]
- 对话目的：[推动情节/展现性格/透露信息]
- 字数：约[X]字

注意参考角色档案中的语言特征：
@02-characters/[角色A档案]
@02-characters/[角色B档案]
```

### 5.2 群戏对话
```
设计一场[X]人的群戏对话：
- 参与者：[列出角色]
- 场景：[描述]
- 主要矛盾/话题：[描述]
- 每个角色要有符合性格的发言
- 通过对话推进[什么情节]
```

### 5.3 内心独白
```
写一段[角色]的内心独白：
- 当前处境：[描述]
- 面对的抉择/困惑：[描述]
- 要展现的性格侧面：[描述]
- 字数：约[X]字
```

## 六、润色优化

### 6.1 段落润色
```
润色以下段落，要求：
- 保持原意不变
- 增强画面感和节奏感
- 语言更加精炼
- 符合[紧张/轻松/悲伤等]的情绪基调

[粘贴原文]
```

### 6.2 对话润色
```
优化以下对话，要求：
- 更符合[角色名]的说话风格（参考@角色档案）
- 更自然流畅
- 潜台词更丰富
- 删除不必要的废话

[粘贴原对话]
```

### 6.3 战斗节奏
```
调整以下战斗场景的节奏：
- 当前问题：[太拖沓/太仓促/缺乏张力等]
- 保持总字数在[X]字左右
- 增强[紧迫感/压迫感/爽感等]

[粘贴原文]
```

## 七、一致性检查

### 7.1 章节检查
```
检查[第X章]的内容：
1. 人物行为是否符合 @02-characters/[角色档案] 的设定
2. 力量表现是否符合 @01-worldbuilding/power-system.md
3. 与前[X]章是否有矛盾之处
4. 时间线是否正确

请列出发现的问题。
```

### 7.2 设定检查
```
检查 @01-worldbuilding/power-system.md 中的力量体系：
1. 是否有逻辑漏洞
2. 各等级之间的跨度是否合理
3. 与科技体系的结合是否自然
4. 建议如何完善
```

### 7.3 伏笔追踪
```
根据 @03-outline/plot-threads.md 中记录的伏笔，
检查第[X]章到第[X]章：
1. 哪些伏笔已经揭示
2. 哪些伏笔需要加强铺垫
3. 是否有遗漏的伏笔需要回收
```

## 八、总结更新

### 8.1 章节总结
```
总结第[X]章的内容：
1. 新增的设定（需要更新到哪个文档）
2. 埋下的伏笔（记录到 plot-threads.md）
3. 人物关系变化（更新角色档案）
4. 主角当前状态（等级、位置、目标）
```

### 8.2 卷册复盘
```
复盘第[X]卷（第[X]章到第[X]章）：
1. 主线推进了多少
2. 支线完成情况
3. 人物成长轨迹
4. 节奏分析（是否有拖沓/仓促）
5. 后续卷需要调整的地方
```
